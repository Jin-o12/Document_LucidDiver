# 📁 문서 관리 및 버전 규칙 (Document Rules)

> 이 문서는 프로젝트 전체 문서의 일관성과 추적 가능성을 보장하기 위한 표준 관리 지침입니다.

---

## 1. 문서 관리 원칙

1. **마크다운(Markdown) 중심**: 모든 기획 및 기술 문서는 `.md` 형식으로 작성합니다.
2. **파일명 한글화**: 팀원 간 직관적인 정보 공유를 위해 파일명은 한글을 허용합니다.
3. **파일명 정규화**: 공백 대신 언더스코어(`_`)를 사용합니다.
   - ✅ `경제_시스템_명세.md`
   - ❌ `경제 시스템 명세.md`
4. **번호 접두사 금지**: 파일명 앞에 숫자 넘버링(01_, 02-1_ 등)을 붙이지 않습니다.

---

## 2. 버전 명명 규칙 (Version Naming)

모든 문서 최하단의 `## 📜 Revision History` 테이블에서 사용되는 버전 체계입니다.

| 버전 | 의미 | 예시 |
|------|------|------|
| **v1.0** | 최초 확정본 | 초안 검수 완료 후 배너 제거 시 |
| **v1.1, v1.2...** | 마이너 업데이트 | 오타 수정, 일부 수치 변경, 문구 보완 |
| **v2.0** | 메이저 업데이트 | 구조 변경, 핵심 로직 개편, 섹션 재편 |

### 이력 기재 규칙
- **단순 요약 금지**: `기획 일부 수정` 같은 한 줄 요약은 허용하지 않습니다.
- **수정 사항 목록화**: 불렛 포인트(`-`)로 변경 내용을 상세 기술합니다.

```markdown
<!-- ❌ 나쁜 예 -->
| 2026-04-29 | v1.1 | 로그인 수정 | 홍길동 |

<!-- ✅ 좋은 예 -->
| 2026-04-29 | v1.1 | - 로그인 타임아웃 변경(30초→60초)<br>- 비밀번호 최소 길이 수정(6자→8자) | 홍길동 |
```

---

## 3. 아카이브 체계 (Archive Management)

### 아카이브 원칙
- **아카이브 시점**: 대규모 수정(메이저 업데이트) 전, 현재 버전을 `Archive/` 폴더에 백업합니다.
- **경로 유지**: 원본 문서의 폴더 경로를 유지하여 파일별로 이력을 격리합니다.

```
원본: 01.기획_문서/경제_시스템_명세.md
백업: Archive/01.기획_문서/경제_시스템_명세/경제_시스템_명세_v1.0.md
```

### 아카이브 예외
- **일일 개발일지**: 매일 새 파일로 생성되므로 Archive 대상에서 제외됩니다.
- **데일리 스크럼**: 팀원별 폴더에 날짜별로 누적 관리됩니다.

---

## 4. 파일명 규칙 상세

### 일반 문서
```
[문서명].md
예: 경제_시스템_명세.md, 로그인_기획서.md
```

### 개발일지
```
개발일지_YYYYMMDD_이름.md
예: 개발일지_20260429_홍길동.md
저장: 03.관리_문서/Scrum_Logs/[이름]/
```

### 데일리 스크럼
```
스크럼_YYYYMMDD.md
예: 스크럼_20260429.md
저장: 03.관리_문서/Scrum_Logs/Daily_Scrum/
```

---

## 5. AI 초안 문서 관리

### 배너 시스템
AI가 생성한 문서에는 상단에 다음 배너가 삽입됩니다:
```markdown
> [!IMPORTANT]
> 이 문서는 AI(Antigravity)가 작성한 초안입니다.
> 기획자/PM의 검토 및 승인 후 이 배너를 제거하면 '확정 사양'으로 인정됩니다.
```

### 상태 정의
| 배너 상태 | 문서 상태 | 구현 가능 여부 |
|-----------|----------|:---:|
| 배너 **있음** | 초안 (참고용) | ❌ |
| 배너 **제거됨** | 확정 사양 | ✅ |
