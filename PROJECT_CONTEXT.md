## 2. Antigravity의 역할 정의

### ✅ 수행하는 역할
- [x] **문서 작성 보조**: 기획 명세서(SSOT), 기술 문서 및 아키텍처 사양 문서 초안 작성
- [x] **스크럼 기록 자동화**: 데일리 스크럼, 개발일지 작성 지원 및 취합
- [x] **코드 및 아키텍처 검토**: Unity 스크립트 구조 및 PUN2 동기화 로직 분석
- [x] **QA 및 테스트 케이스 설계**: 세션 탈출, 인벤토리 소실, PUN2 동기화 테스트 시나리오 작성
- [x] **발표 자료 보조**: 발표 대본 정리, PPT 구성 슬라이드 보조

### 🚫 수행하지 않는 역할 (금지 사항)
- 기획자/PM의 명시적인 검토 및 배너 제거(`[!IMPORTANT]` 제거) 없이 임의로 사양을 확정하지 않음.
- 유니티 프로젝트 소스 코드를 PM 승인 없이 임의로 영구 수정하거나 삭제하지 않음.

## 3. 문서 관리 규칙

### 작업 범위
- **쓰기 가능 영역**:
  - `01.기획_문서/`
  - `02.기술_문서/`
  - `03.관리_문서/` (하위 `Scrum_Logs/`, `Daily_Scrum/`)
  - `04.QA_및_테스트/`
  - `05.발표_및_시연/`
- **읽기 전용 영역**:
  - `Archive/`
  - `Templates/`
- **프로젝트 소스 코드**:
  - 필요한 스크립트 작성 및 구조 분석은 허용되나, 직접 커밋 전 검토 필수.

### 문서 확정 프로세스
```
AI 초안 작성 (상단 [!IMPORTANT] 배너 삽입) ➡️ PM 검토 및 피드백 ➡️ 승인 후 배너 제거 ➡️ '확정 사양' 인정
```

### 버전 관리
- 모든 문서는 최하단에 `## 📜 Revision History` 테이블을 유지하며 마이너/메이저 변경 이력을 상세 기재.
- 메이저 업데이트 시 기존 버전은 `Archive/[폴더명]/[파일명]_v[버전].md`로 백업 이관.

├── 📁 03.관리_문서/                  ← 스크럼 로그, 팀원 개발일지 및 로드맵

├── 📁 05.발표_및_시연/               ← PPT 구성안 및 발표 대본

├── 📁 Templates/                    ← 문서 템플릿 양식 모음

└── 📄 DOCUMENT_RULES.md             ← 문서 명명 및 버전 규칙
