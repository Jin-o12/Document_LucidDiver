## 0. 개발 환경 소개

### IDE: VS Code + Gemini Code Assist
Antigravity는 **VS Code (Visual Studio Code)** 에디터 안에서 동작합니다.
별도의 웹 브라우저나 외부 프로그램이 아니라, **코드 에디터 안에 내장된 채팅 패널**에서 자연어로 대화하며 작업합니다.

#### 설치 방법
1. [VS Code](https://code.visualstudio.com/) 설치
2. 확장 프로그램에서 **Gemini Code Assist** 검색 후 설치
3. Google 계정으로 로그인
4. 좌측 사이드바의 Gemini 아이콘(✨) 클릭 → 채팅 패널 열기

#### 작업 흐름
```
┌──────────────────────────────────────────────┐
│  VS Code                                     │
│  ┌────────────────────┬─────────────────┐    │
│  │   파일 탐색기 +     │  Antigravity    │    │
│  │   에디터 영역       │  채팅 패널      │    │
│  │                    │                 │    │
│  │  여기서 파일을      │  여기서 자연어로  │    │
│  │  열어보고 편집      │  대화하며 지시    │    │
│  │                    │                 │    │
│  └────────────────────┴─────────────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  터미널 (Antigravity가 명령 실행)      │    │
│  └──────────────────────────────────────┘    │
└──────────────────────────────────────────────┘
```

> **핵심**: ChatGPT처럼 웹에서 대화하는 게 아니라, **내 프로젝트 폴더가 열린 에디터 안에서 대화**하기 때문에 AI가 파일을 직접 읽고, 만들고, 수정할 수 있습니다.

## 1. 기본 개념

### Antigravity란?
Antigravity는 Google DeepMind에서 개발한 AI 코딩 어시스턴트로, 일반적인 챗봇과 달리 **실제 파일 시스템에 접근**할 수 있습니다.

| 기능 | 설명 |
|------|------|
| 📄 **파일 읽기/쓰기** | 프로젝트 내 파일을 직접 열어보고, 새 파일 생성·수정 가능 |
| 💻 **터미널 명령 실행** | git, npm 등 커맨드 라인 명령을 대신 실행 |
| 🌐 **웹 검색** | 필요한 정보를 인터넷에서 검색하여 답변에 활용 |
| 🖼️ **이미지 생성** | 간단한 에셋이나 UI 목업 이미지를 생성 |
| 🔌 **MCP 연동** | Unity, DB 등 외부 도구와 연결하여 데이터 접근 |

### Co-PM으로 쓴다는 건?
AI를 단순 코드 작성 도구가 아니라, **프로젝트 문서를 함께 관리하는 부PM**으로 활용하는 것입니다:

| 전통적 AI 활용 | Co-PM으로서의 AI |
|---------------|-----------------|
| "이 함수 짜줘" | "이 기능의 기획 명세서를 작성해줘" |
| "버그 고쳐줘" | "QA 테스트 시나리오를 만들어줘" |
| "코드 리뷰해줘" | "오늘 스크럼 일지를 정리해줘" |

### 핵심 원칙: SSOT (Single Source of Truth)
- 프로젝트의 "진짜 현재 상태"를 담은 문서가 딱 **한 곳**에만 존재해야 합니다.
- AI가 문서를 갱신할 때 항상 이 원칙을 따르게 해야 합니다.

## 2. Antigravity 세부 활용법

### 2.1 `@` 멘션 — 파일을 AI에게 보여주기

채팅에서 `@`를 입력하면 프로젝트 내 파일을 참조할 수 있습니다.
AI가 해당 파일의 내용을 직접 읽고 이해한 상태에서 답변합니다.

```
@[파일경로] 이 파일을 분석해서 요약해줘
```

**활용 예시:**
| 상황 | 프롬프트 |
|------|---------|
| 기존 문서 수정 | `@[01.기획_문서/로그인_시스템.md] 여기에 소셜 로그인 항목 추가해줘` |
| 코드 분석 | `@[src/main.py] 이 코드의 구조를 기술 문서로 작성해줘` |
| 템플릿 참고 | `@[Templates/TEMPLATE_기획_명세서.md] 이 양식으로 결제 시스템 명세서 써줘` |
| 여러 파일 비교 | `@[파일A] @[파일B] 이 두 문서의 차이점을 정리해줘` |

> **팁**: 파일을 열어놓은 상태에서 대화하면, AI가 현재 열린 파일의 컨텍스트도 자동으로 인식합니다.

### 2.2 자연어 대화 — 말하듯이 지시하기

Antigravity에게는 코드나 명령어가 아닌 **한국어 자연어**로 말하면 됩니다.

**❌ 이렇게 할 필요 없음:**
```
create file ./01.기획_문서/login_spec.md --template=기획_명세서
```

**✅ 이렇게 하면 됨:**
```
로그인 시스템 기획서를 기획_명세서 템플릿으로 만들어서
01.기획_문서 폴더에 저장해줘
```

### 2.3 대화 컨텍스트 — AI의 기억

Antigravity는 **하나의 대화 세션 안에서** 이전 대화 내용을 기억합니다.

```
나: "로그인 시스템 명세서를 작성해줘"
AI: (명세서 작성)
나: "거기에 소셜 로그인 항목도 추가해"   ← "거기"가 뭔지 AI가 알고 있음
AI: (해당 명세서에 추가)
나: "그 문서를 한국어로 요약해줘"           ← "그 문서"도 알고 있음
```

> **주의**: 새 대화를 시작하면 이전 맥락이 사라집니다.
> 이때 `@[PROJECT_CONTEXT.md]`를 멘션하면 프로젝트 맥락을 빠르게 복원할 수 있습니다.

### 2.4 작업 승인 시스템

Antigravity가 파일을 수정하거나 명령을 실행할 때, 위험할 수 있는 작업은 **사용자 승인을 요청**합니다.

| 작업 유형 | 자동 실행 | 승인 필요 |
|-----------|----------|----------|
| 파일 읽기 | ✅ | |
| 새 파일 생성 | ✅ | |
| 기존 파일 수정 | ✅ | |
| 안전한 터미널 명령 (ls, dir 등) | ✅ | |
| 파일 삭제, 외부 설치 등 | | ⚠️ |

승인 요청이 뜨면 내용을 확인하고 `승인` 또는 `거절`을 선택하면 됩니다.

### 2.5 효과적인 지시 팁

| 팁 | 예시 |
|----|------|
| **구체적으로** | ❌ "문서 정리해줘" → ✅ "01.기획_문서의 모든 .md 파일 목록을 표로 정리해줘" |
| **경로를 명시** | ❌ "저장해줘" → ✅ "03.관리_문서/Scrum_Logs/ 에 저장해줘" |
| **양식 지정** | ❌ "기획서 써줘" → ✅ "Templates/TEMPLATE_기획_명세서.md 양식으로 써줘" |
| **수정 범위 한정** | ❌ "문서 고쳐줘" → ✅ "3번 섹션의 데이터 구조 표만 업데이트해줘" |
| **역할 부여** | ❌ "이거 봐줘" → ✅ "QA 담당자 관점에서 이 기획서의 빈틈을 찾아줘" |

## 3. 문서 자동 생성

### 기획 명세서 생성 요청하기
```
Templates/TEMPLATE_기획_명세서.md 양식을 참고해서
[기능명]에 대한 기획 명세서를 01.기획_문서/ 에 작성해줘.
```

### 기술 문서 생성 요청하기
```
[기능/모듈명]의 기술 구현 명세서를 02.기술_문서/ 에 작성해줘.
현재 코드 구조를 분석해서 실제 구현과 일치하게 작성해.
```

### 여러 문서 일괄 생성하기
```
아래 목록의 기획 명세서를 일괄 생성해줘. 
양식은 Templates/TEMPLATE_기획_명세서.md를 따르고,
각각 01.기획_문서/ 에 개별 파일로 저장해:
1. 로그인 시스템
2. 채팅 기능
3. 상점 시스템
```

## 4. 스크럼 자동화

### 데일리 스크럼 기록
매일 작업 완료 후 아래처럼 요청합니다:
```
오늘 데일리 스크럼 정리해줘.
- 완료: [작업 내용 나열]
- 진행 중: [작업 내용 나열]  
- 블로커: [있으면 작성]
03.관리_문서/Scrum_Logs/Daily_Scrum/ 에 저장해줘.
```

### 개인 개발일지 자동화
```
Templates/TEMPLATE_개발일지.md 양식으로
오늘 개발일지를 작성해줘.
내용: [오늘 한 작업 간략히 나열]
03.관리_문서/Scrum_Logs/[이름]/ 에 저장해.
```

### 주간 요약 리포트
```
이번 주(월~금) 데일리 스크럼 기록을 종합해서
주간 요약 리포트를 만들어줘.
진행률, 완료 항목, 잔여 이슈를 표로 정리해.
```

## 5. AI 초안 검수 프로세스

실전 프로젝트에서 검증된 3단계 프로세스:

```
┌─────────────────────────────────────────────┐
│  Step 1: AI 초안 생성                         │
│  → 문서 상단에 [!IMPORTANT] 배너 자동 삽입     │
├─────────────────────────────────────────────┤
│  Step 2: PM/기획자 검토                       │
│  → 내용 확인, 수정 요청, 승인                  │
├─────────────────────────────────────────────┤
│  Step 3: 배너 제거 = 확정                     │
│  → 이제 이 문서 기반으로 구현 가능              │
└─────────────────────────────────────────────┘
```

### 배너 예시
AI가 초안을 작성하면 자동으로 이 배너가 문서 최상단에 붙습니다:
```markdown
> [!IMPORTANT]
> 이 문서는 AI(Antigravity)가 작성한 초안입니다.
> 기획자/PM의 검토 및 승인 후 이 배너를 제거하면 '확정 사양'으로 인정됩니다.
```

### AI에게 검수 규칙 적용하기
프로젝트 시작 시 한 번만 말해주면 됩니다:
```
앞으로 네가 새 문서를 생성할 때마다 상단에 [!IMPORTANT] 배너를 넣어줘.
내가 직접 배너를 지우기 전까지는 초안 상태야.
```

## 6. Unity MCP 연동 (선택)

Unity 프로젝트가 있다면, MCP(Model Context Protocol)를 통해 AI가 유니티 에디터를 **읽기 전용**으로 접근하여 역기획(Reverse Engineering) 기반 문서를 작성할 수 있습니다.

### 연동 시 규칙
```
유니티 MCP 연동 규칙:
1. 읽기 전용(Read-only)으로만 접근할 것
2. 유니티 프로젝트의 코드나 에셋을 직접 수정하지 말 것
3. 문서 작성은 오직 이 문서 저장소 내에서만 할 것
4. 실제 구현된 기능만 문서화하고, 미구현 기능은 포함하지 말 것
```

### 역기획 문서 생성 요청
```
유니티 MCP를 통해 [스크립트명/씬명]을 분석하고,
실제 구현 내용을 기반으로 기술 명세서를 작성해줘.
기존 기획서가 아니라 코드 실체가 진실이야.
```


### 🟡 새 기능 기획 시
```
[기능명] 기획 명세서를 작성해줘.

- 01.기획_문서/ 에 저장해
```

### 🟡 문서 갱신 시
```
@[문서 파일 경로] 이 문서를 현재 상태에 맞게 업데이트해줘.
변경 사항은 Revision History에 기록해.
```

### 🔴 대규모 정리 시
```
01.기획_문서/ 전체 문서를 검토해서:
1. 오래된 내용이 있는 문서 목록을 알려줘
2. 현재 구현과 불일치하는 부분을 표시해줘
3. 누락된 문서가 있다면 제안해줘
```

### 🔴 발표 준비 시
```
이 프로젝트의 최종 발표 대본을 작성해줘.
- 슬라이드 [N]장 기준, 총 [N]분 발표
- 슬라이드별 예상 시간 배분 포함
- 05.발표_및_시연/ 에 저장해
```

### 🟣 문서 인벤토리 자동 생성
```
01~05 폴더의 모든 .md 파일을 스캔해서
README.md에 전체 문서 인벤토리(목록 + 링크)를 업데이트해줘.
카테고리별로 분류하고, 각 문서의 버전과 최종 수정일도 표시해.
```

### 🟣 기획서 가이드 참조
```
@[SPEC_WRITING_GUIDE.md] 이 가이드를 따라서
[기능명] 기획 명세서를 작성해줘. 01.기획_문서/ 에 저장해.
```

## 8. 트러블슈팅 FAQ

### Q: AI가 컨텍스트를 잊어버려요
**A:** 새 대화를 시작할 때마다 `@PROJECT_CONTEXT.md`를 멘션하세요. 이 파일이 AI의 "기억 복원 포인트" 역할을 합니다.

### Q: AI가 아직 없는 기능을 문서에 넣어요
**A:** `PROJECT_CONTEXT.md`에 금지 사항을 명시하세요:
```
🚫 실제 구현되지 않은 기능은 문서에 포함하지 마세요.
```

### Q: 문서가 너무 많아져서 관리가 힘들어요
**A:** 주기적으로 AI에게 정리를 요청하세요:
```
현재 문서 목록을 정리하고, 중복이나 오래된 문서를 Archive로 이동해줘.
README의 문서 인벤토리도 업데이트해.
```

### Q: 팀원이 여러 명인데 충돌이 나요
**A:** 작업 영역을 명확히 나누세요:
- AI는 `01~05` 폴더의 `.md` 파일만 담당
- 팀원은 실제 코드만 담당
- 둘이 같은 파일을 동시에 수정하지 않도록 스크럼에서 조율
