# 1. 플레이어 캐릭터

## 1.1. 오브젝트 구성

- 플레이어 캐릭터 오브젝트는 이하의 하위 오브젝트로 구성된다.
    - 위치 기준점 (Transform): 플레이어의 위치 및 방향(Y 위, Z 정면) 정보를 계산하는 기준점.
    - 몸통 (Collider): 플레이어↔적, 플레이어↔장애물 간의 접촉 상호작용 발동 여부를 판정하는 기준 영역.
        - 장애물 판정을 위해 Trigger 설정을 비활성화한다.
    - 상호작용 구역 (Collider): 플레이어↔상호작용 가능 오브젝트(무장, 회수품) 간의 상호작용 UI 출력 여부를 판정하는 기준 영역.
        - Trigger 설정을 활성화하여 사용한다.
    - 손 (Transform): 무장 아이템 장착 시 무장 오브젝트가 배치되는 위치 기준점.
        - 공격 방향(vector3): 위치 기준점을 시작점, 손을 끝점으로 하는 방향 벡터.
            - 높이(Unity 좌표계의 Y축) 값을 0으로 초기화한 후 길이 Normalize를 적용하여 사용한다.

## 1.2. 플레이어 상태

- 플레이어 캐릭터는 bool 또는 enum으로 표기되는 상태 값을 갖는다.
    - 재장전 중(`isReloading` : bool): 플레이어가 탄창을 전부 사용하여 재장전 상태인지 확인
    - 생존 상태(`livingState` : enum): 플레이어의 메인 캐릭터 상태를 확인
        - 0: idle `기본`
        - 1: escape`탈출 중`
        - 2: gameover `강제각성`
        - 탈출 조건 우선 순위는 enum 오름차순으로 판정한다(탈출 중 → 강제각성 순서로 처리)

## 1.3. 플레이어 조작

### 1.3.1. 이동

- 화면 왼쪽의 가상 조이스틱을 드래그하여 이동 명령을 입력한다.
    - PC (키보드+마우스): WASD 및 화살표(↑←↓→) 조작을 사용해 이동 명령을 입력
    - 모바일(터치패드): 화면 왼쪽의 가상 조이스틱 드래그
        - 가상 조이스틱의 드래그 시작 지점과 드래그 유지 지점 사이의 방향 벡터(Vector2, 길이 Normalize 적용) 값만큼 이동 명령을 전달한다.
- 입력된 이동 명령 값에 플레이어의 이동 속도 및 이동 명령 입력 시간(Time.deltaTime)을 곱한 값만큼 플레이어 캐릭터의 실제 이동을 처리한다.

### 1.3.2. 기본 공격

- 조준 방향 입력
    - PC (키보드+마우스): UI Raycast Target이 적용되지 않는 영역을 마우스 왼쪽 버튼 클릭으로 홀드하여, 플레이어 위치 기준점의 UI Canvas 상 위치와 클릭 지점 사이의 방향 벡터를 기본 공격 명령 값으로 입력한다.
        - 입력된 기본 공격 명령에 따라 가상 조이스틱 드래그 애니메이션을 적용한다.
    - 모바일/PC: 화면 오른쪽의 가상 조이스틱을 드래그하여 기본 공격 명령을 입력한다.
        - 가상 조이스틱의 드래그 시작 지점과 드래그 유지 지점 사이의 방향 벡터(Vector2, 길이 Normalize 적용)을 기본 공격 명령 입력 값으로 전달한다.
- 입력된 기본 공격 명령 벡터를 높이(Y) 값이 0인 Vector3으로 환산한 후, 플레이어 캐릭터의 공격 방향 벡터를 기본 공격 명령 벡터 방향으로 회전시킨다.
- 플레이어 캐릭터 오브젝트의 위치 기준점에서 공격 방향 벡터 방향으로 무장의 발사 가능 사거리 길이만큼 공격 Ray를 출력한다.
- 공격 Ray가 적의 피격 범위 Collider에 접촉하면 무장 아이템 오브젝트의 총구 위치에서 기본 공격 루틴을 실행한다.
    - 이하의 경우 중 어느 것에 해당하는 상태에서는 기본 공격 루틴이 실행되지 않는다.
        - 장착 중인 무장이 없는 경우
        - `재장전 중==true` 상태인 경우
        - 플레이어가 보유한 마나가 기본 공격에 필요한 마나 미만인 경우
        - 이전 기본 공격 루틴 1회 실행 후 경과한 시간이 발사 간격 시간 미만인 경우

### 1.3.3. 오브젝트 상호작용

- 플레이어 캐릭터의 상호작용 구역 Collider에 상호작용 가능한 오브젝트의 Collider가 접촉하면 상호작용 버튼이 오브젝트의 정해진 위치에 출력된다.
- 상호작용 버튼이 출력된 상태에서 이하의 조작을 통해 해당 오브젝트와 상호작용을 실행한다.
    - PC: Input System에 등록한 상호작용 키를 입력
    - PC/모바일: 상호작용 버튼을 터치(PC 환경에서는 마우스 클릭)

## 1.4. 마나

- 월드 진입 시 최대 100을 보유하며, 매 초마다 정해진 `초당 마나 회복` 수치만큼 자동으로 회복된다.

## 1.5. 지원 캐릭터

- 메인 캐릭터 이외에 추가로 편성 가능한 캐릭터.
- 호출 스킬을 사용할 수 있음.

### 1.5.1. 심상 공명

- 플레이어 캐릭터의 심상 공명 태그가 정해진 조합을 달성한 경우에 획득할 수 있는 패시브 효과.
    - 심상 공명 태그는 `역할 태그`, `기물 태그`, `소속 태그`로 구분된다.
    - 각 메인 캐릭터 및 지원 캐릭터는 1개 이상의 심상 공명 태그를 가진다.
        - 심상 공명 태그를 2개 이상 보유하는 경우, 역할/기물/소속 태그 구분 중에서는 1개까지만 가질 수 있다.

# 2. 맵 설정

- 본 게임의 맵은 2.5D 시점으로 작업한다.
    - 3D 신에서 X,Z의 2개 좌표만을 사용하여 캐릭터(플레이어, 적)이 이동하는 레벨을 구성한다.

## 2.1. 카메라 설정

- 메인 카메라에 Cinemachine을 적용하여, 플레이어 메인 캐릭터 Transform을 Follow 및 LookAt 대상으로 지정한다.
    - 카메라가 플레이어의 위치 이동을 추적하여 동일한 속도로 이동한다.

## 2.2. 시작 지점

- 각 맵마다 정해진 위치에 시작 지점을 설정한다.
    - 시작 지점에는 플레이어 캐릭터 오브젝트를 출력하는 출력 기준점 Transform 오브젝트가 배치된다.
- 플레이어가 맵에 진입하면 플레이어 캐릭터 오브젝트가 시작 지점에 출력된다.
    - 시작 지점의 출력 기준점 위치와 플레이어 캐릭터의 위치 기준점이 일치하도록 출력한다.

## 2.3. 탈출 지점

- 각 맵에 정해진 개수의 탈출 지점을 설정한다.
    - 탈출 지점에는 플레이어 캐릭터 오브젝트의 탈출 여부를 감지하는 탈출 구역 (Collider) 오브젝트가 배치된다.
        - 탈출 구역 Collider의 Trigger 설정을 활성화한다.
    - 탈출 지점에 데이터 테이블과 연동되는 스크립트를 컴포넌트로 연결하여, 탈출 가능 조건 및 탈출 가능한 플레이어 수를 설정한다.
- 플레이어의 몸통 Collider와 탈출 지점의 탈출 구역 Collider가 접촉하면 OnTriggerEnter 이벤트를 통해 플레이어 탈출 판정을 실행한다.
    - 탈출 판정 실행 시 정해진 채널링 시간 동안 탈출 구역 Collider와 접촉 상태를 유지해야 한다.
        - 채널링 시간 동안 플레이어 상태를 `탈출 중` 상태로 한다.
        - 채널링 없이 즉시 탈출하는 탈출 지점은 채널링 시간을 0으로 입력하여 구현한다.
        - 채널링 시간 중 탈출 구역 Collider에서 이탈하면 OnTriggerExit 이벤트에 의해 탈출 판정이 취소된다.
        - 탈출 구역 Collider에 접촉한 상태를 유지하며 채널링 시간 경과 시 탈출 성공 이벤트를 실행한다.

## 2.4. 탈출 판정

- 맵 플레이 중 이하의 어느 조건 중 1개 이상이 만족되면 탈출 판정을 실행한다.
    - 플레이어 캐릭터의 보유 체력이 0이 됨 → `강제 각성`  상태로 탈출 판정을 실행한다. (탈출 실패)
    - 플레이어가 탈출 지점에 도착 → `탈출 중`  상태를 채널링 시간 동안 유지 → 채널링 시간 경과 후 탈출 판정을 실행한다. (탈출 성공)
- 탈출 판정 조건은 각 플레이어마다 개별로 계산한다.
- 탈출 판정을 실행하여 탈출 성공 이벤트 실행 시 탈출 성공 이벤트가 실행된 시점의 보유 체력, 처치한 적 수, 플레이 시간, 인벤토리 리스트를 저장하여 결과 창에 출력한다.
    - 인벤토리 내에 저장된 아이템은 아이템 종류 및 탈출 성공/실패 판정에 따라 이하와 같이 정산한다.
    
    | 아이템 종류 | 각성 보존 슬롯 보관 | 탈출 성공 | 탈출 실패 |
    | --- | --- | --- | --- |
    | 기본 무장 |  | 유지됨 | 유지됨 |
    | 특화 무장 | ○ | 유지됨 | 유지됨 |
    |  | × | 유지됨 | 소멸됨 |
    | 드림 포지드 무장 |  | ‘무장 파편’으로 분해 | 소멸됨 |
    | 그 외 회수품 | ○ | 유지됨 | 유지됨 |
    |  | × | 유지됨 | 소멸됨 |

## 2.5. 시간 차감 가속

- 맵 내의 정해진 구역에 플레이어가 진입해 있는 경우 제한 시간 차감 속도가 정해진 시간 배율로 가속된다.
    - 가속 구역 Collider 볼륨에 플레이어 몸통 Collider가 접촉하고 있는 동안 OnTriggerStay 코드를 통해 시간 배율을 적용한다.

# 3. 아이템

## 3.1. 아이템 생성 및 상호작용

- 월드 신이 생성되는 시점에 월드 신마다 지정된 위치에 아이템이 생성된다.
- 아이템과 플레이어가 상호작용하면 해당 플레이어의 인벤토리에 상호작용한 아이템을 등록한다.
    - 인벤토리 등록 처리 후 월드 맵의 아이템 오브젝트를 비활성화하여 아이템 ObjectPool로 되돌린다.

## 3.2. 무장

- 플레이어가 장착하여 적을 공격하는 데에 사용하는 아이템.

### 3.2.1. 오브젝트 구성

- 무장 아이템 오브젝트는 이하의 하위 오브젝트로 구성된다.
    - 손잡이(transform): 플레이어 캐릭터가 무장 장착 시 플레이어 캐릭터의 손 오브젝트와 무장 아이템 오브젝트의 위치 및 방향을 일치시키는 기준점.
    - 총구(transform): 기본 공격 루틴이 출력되는 위치 및 방향 기준점.
    - 공격 이펙트 (particle system): 기본 공격 루틴 실행 시 출력되는 공격 이펙트 오브젝트.

### 3.2.2. 무장 획득

- 월드 맵에 생성된 무장 아이템은 플레이어가 오브젝트 상호작용을 통해 획득할 수 있다.
- 획득한 무장은 인벤토리에 보관된다.

### 3.2.3. 무장 장착

- 상호작용으로 획득한 무장 아이템은 인벤토리에서 장착할 수 있다.
    - 인벤토리에서 무장 아이템 아이콘을 무장 슬롯에 드래그하여 장착한다.
        - 드래그한 슬롯에 장착 중인 무장이 없는 경우: 드래그한 무장 아이템을 그대로 빈 무장 슬롯에 장착한다.
        - 드래그한 슬롯에 장착 중인 무장이 있는 경우
            - 드래그한 슬롯에 장착 중인 무장의 장착을 해제하여, 드래그한 슬롯을 장착 중인 무장이 없는 상태로 변경한다.
            - 드래그한 슬롯에 드래그한 무장 아이템을 장착한다.

### 3.2.4. 기본 공격 루틴

- 총알 발사 루틴은 이하의 순서에 따라 실행된다.
    - 기본 공격 1회에 필요한 마나 수량만큼 플레이어 보유 마나가 감소한다.
    - 장착 중인 무장에 지정된 공격 이펙트가 출력된다.
    - 장착 중인 무장에 지정된 공격 범위 내에 해당하는 적에게 명중 판정 이벤트를 발생시킨다.
        - 명중 판정 이벤트는 현재 몽막 피해 → 현재 체력 피해 → 사망 처리 → 심상핵 노출 상태 순서로 실행한다.

### 3.2.5. 재장전

- 기본 공격 루틴은 정해진 탄창 용량 수치만큼만 연속으로 실행할 수 있다.
    - `현재 탄창`(int)과 `재장전 중`(bool)을 내부 변수로 지정한다.
        - 예외 처리: 재장전 루틴을 실행하지 않는 무장은 현재 탄창 값을 0 미만인 값으로 지정한다.
    - 기본 공격 루틴을 실행할 때마다 실행된 횟수만큼 현재 탄창 값이 감소한다.
- 현재 탄창이 0이 된 시점에 이하의 순서에 따라 재장전 루틴을 실행한다.
    - `재장전 중=true` 상태가 된다.
    - `재장전 시간`만큼 대기한다.
    - `현재 탄창` 값을 `탄창 용량` 수치만큼으로 초기화한다.
    - `재장전 중=false` 상태가 된다.

## 3.3. 방어구

- 플레이어가 장착하여 방어막/방어력 수치를 얻는 아이템.
    - 방어구 아이템을 장착한 플레이어는 방어구에 설정된 방어막 수치 합계만큼의 `최대 방어막` (int) 값을 내부 변수로 가진다.
        - 추가로, 현재 시점의 방어막 수치를 가리키는 `현재 방어막` (int) 및 방어막 재생 효과 발동 여부 판정에 사용되는 `최근 피격 시점` (time)을 내부 변수로 가진다.
        - 현재 방어막 수치가 최대 방어막 수치 미만인 상태에서 마지막 피격 판정 후 `방어막 재생 대기시간` 이상 시간 경과 시, 현재 방어막이 1초마다 `초당 방어막 회복량` 만큼 증가한다. (최대 방어막 값까지)
        - 현재 방어막 수치가 1 이상인 상태에서 플레이어 캐릭터가 적 캐릭터 공격에 피격 시 피격 경직 판정이 발생하지 않는다.
    - 방어구 아이템을 장착한 플레이어는 방어구에 설정된 방어력 수치 합계만큼의 `총 방어력` (float) 값을 내부 변수로 가진다.
        - 총 방어력은 `최대 방어력`  상수 값 이하로 적용한다. (최대 방어력 값은 밸런스 기획에서 설정)

### 3.3.2. 정신 과부하 페널티

- 각 방어구 아이템에는 정신 과부하 페널티가 존재한다.
    - 정신 과부하 페널티로는 `마나 회복 속도 감소` 및 `이동 속도 감소` 가 존재한다.
        - 마나 회복 속도 감소 및 이동 속도 감소 페널티는 합연산으로 적용되며, 각각  `마나 회복 감소 최대값` 및 `이동 속도 감소 최대값` 상수 이하로 적용한다. (각각의 상수 값은 밸런스 기획에서 설정)
        - 마나 회복 증가 또는 이동 속도 증가 버프를 적용할 경우 해당 버프 수치만큼의 음수(-) 값으로 입력하여 적용할 수 있다.

# 4. 적

## 4.1. 오브젝트 구성

- 적 캐릭터 오브젝트는 이하의 하위 오브젝트로 구성된다.
    - 위치 기준점 (Transform): 적의 위치 및 방향(Y 위, Z 정면) 정보를 계산하는 기준점.
    - 몸통 (Collider): 적과 다른 구성 요소 사이의 접촉 상호작용 발동 여부를 판정하는 기준 영역.
        - 장애물 판정을 위해 Trigger 설정을 비활성화한다.

## 4.2. 적 상태 제어

- 적 캐릭터의 행동 패턴은 이하의 `enemyState` (enum) 상태에 따라 제어한다.
    - idle `기본`
    - detect `추적`
    - attack `공격`
    - hit `피격`
    - dead `사망`
- 적 캐릭터 오브젝트에 플레이어 추적 패턴을 설정한다.
    - 플레이어 추적 패턴은 시야 거리만큼의 반지름과 시야 각도만큼의 사이각을 가지는 부채꼴 영역(이하 `추적 영역`) 내에서 작동한다.
        - 추적 영역은 위치 기준점 정면 방향에서 좌우(각각 -,+ 방향)로 시야 각도 값의 절반만큼 벌린 영역이 된다.
    - 적 캐릭터의 추적 영역 내에 플레이어가 진입하면 적의 추적 상태가 `추적` 으로 변경된다.
        - 플레이어 진입 판정은 매 프레임마다 갱신된다.
        - 플레이어 진입은 플레이어의 위치 기준점이 추적 영역 내에 위치하는지 여부로 판정한다.
            - 추적 영역 내에 플레이어의 위치 기준점 오브젝트가 없으면 `대기` 상태로 돌아간다.
        - `강제 각성` 상태나 `탈출 중` 상태인 플레이어는 진입 판정을 실행하지 않는다.
        - `추적` 상태인 적 캐릭터는 추적 영역 내의 플레이어 중 거리가 가장 가까운 캐릭터를 향해 이동한다.
            - 적의 위치 기준점 정면 방향을 플레이어 위치 방향으로 Rotate시킨다. (보간 적용)
    - 추적 영역 내에 적 캐릭터와의 거리 값이 공격 사거리 이하인 플레이어 캐릭터가 있으면 적의 추적 상태가 `공격` 으로 변경된다.
        - 공격 상태인 적은 위치 기준점 사이의 거리가 가장 가까운 플레이어 1명에게 기본 공격 루틴을 실행한다. (각 적에게 지정된 기본 공격 스킬을 발동)
            - 공격 애니메이션을 재생하고, 피해 판정 이벤트가 입력된 시점에 공격 대상 플레이어에게 체력 감소 처리를 발생시킨다.
                - 공격 시작 시점부터 피해 판정 이벤트가 입력된 시점까지의 시간 간격을 공격 쿨타임 데이터로 관리한다.
        - 공격 루틴 실행 후 공격 간격 시간 동안 대기한다.
            - 대기 중 추적 영역 내의 플레이어 캐릭터를 검색하여 추적 상태를 갱신한다.
- 플레이어가 적 캐릭터를 공격 시 피격 판정이 발생한다.
    - 피해 계산 후 남은 보유 체력 수량을 체크한다.
        - 피격된 적 캐릭터의 보유 체력이 0 이하가 된 경우 적 사망 처리를 실행한다.
        - 피격된 적 캐릭터의 보유 체력이 1 이상인 경우 슈퍼아머 상태 여부를 체크한다.
            - 슈퍼아머 상태인 경우 적 캐릭터 상태를 피격 판정 시점의 상태로 유지한다.
            - 슈퍼아머 상태가 아닌 경우 적 캐릭터 상태를 `enemyState=hit` (피격)으로 변경한다.
                - 피격 상태로 변경 시 플레이어 추적 패턴을 해제하고 피격 애니메이션을 재생한다(경직 상태)
                - 피격 애니메이션 재생 길이만큼 시간 경과 후 적 캐릭터 상태를 `enemyState=idle` (기본)으로 변경하고 플레이어 추적 패턴을 다시 실행한다.
                    - 다음 프레임에 플레이어와 적 사이의 거리에 따라 enemyState가 detect 또는 attack으로 갱신된다.

## 4.3. 몽막 / 심상핵

- 적 캐릭터는 몽막 게이지를 가진다.
    - 월드에 스폰된 적은 ‘현재 몽막’ (int) 값을 내부 변수로 가지며, 월드에 활성화된 시점에 몽막 게이지 값만큼의 현재 몽막을 보유한 상태로 스폰된다.
- 몽막 게이지가 1 이상인 적 캐릭터는 `심상핵 노출` (bool) 상태를 내부 변수로 가진다.
    - 심상핵 노출 값은 (현재 몽막 값이 0 이하인가)를 기록하며, 초기 상태에서는 false로 설정된다.
- 플레이어의 공격(무장 기본 공격 및 스킬)에는 각각 정해진 `몽막 공격력` 수치를 가진다.
    - 플레이어의 공격이 적에게 히트 시 몽막 공격력만큼 대상 적의 현재 몽막 값이 감소한다.
    - 현재 몽막 피해 계산은 보유 체력 피해 계산보다 먼저 적용되며, 별개의 메소드를 통해 동작한다.
- 플레이어의 기본 공격 및 스킬에 피격되어 적 캐릭터의 몽막 게이지가 0 이하가 되면 `심상핵 노출=true` 상태가 된다.
    - 심상핵 노출 상태 판정은 현재 몽막 피해 계산과 별개의 메소드를 통해 동작하며, 보유 체력 피해 계산보다 나중에 적용한다.
- 심상핵 노출 상태인 적을 플레이어가 기본 공격 및 스킬로 공격 시, 적의 보유 체력이 감소하는 피해량이 적 종류에 따라 정해진 심상핵 치명타 배율만큼 증가된다.
    
    $기본 피해량 × (1 + 심상핵 치명타 배율)$
    
- `심상핵 노출=true` 상태로 몽막 재생 시간만큼 시간 경과 시 이하의 몽막 재생 처리를 실행한다.
    - 현재 몽막 값을 몽막 게이지 값으로 초기화한다.
    - `심상핵 노출=false` 상태가 된다.

# 5. 전투

## 5.1. 피해 계산

- 플레이어 및 적 캐릭터는 보유 체력(int)을 내부 변수로 가진다.
    - 보유 체력은 각 캐릭터가 월드에 등장한 시점에 최대 체력 값만큼의 초기값을 부여받는다.
- 플레이어가 발사한 총알이 적에게 명중 판정 발생 시 적의 보유 체력이 감소한다.
- 적이 공격 루틴을 실행하여 플레이어의 보유 체력을 감소시킨다.
- 공격받은 대상의 보유 체력이 감소하는 수량은 이하의 수식을 사용하여 계산한다.
    
    $$
    체력감소_B=공격력_A×공격 배율_A×(1-방어력_B)
    $$
    
    - A:공격한 대상, B: 공격받은 대상.
    - 기본 공격은 [공격 배율]을 100%로 계산한다.
- 현재 방어막이 1 이상인 플레이어에게 보유 체력 감소 판정이 발생한 경우, 현재 방어막 값이 보유 체력 값보다 우선하여 감소한다.
    - 체력 감소량이 보유 방어막 값을 초과하는 경우, 그 넘은 분량만큼 보유 체력 값을 감소시킨다.

## 5.2. 체력 소진 시 처리

- 공격받은 대상의 보유 체력이 0 이하가 되면 대상의 종류에 따라 사망 처리를 실행한다.

### 5.2.1. 플레이어 - 강제 각성

- 플레이어의 livingState 상태를 gameover `강제 각성` 으로 변경한다.
- 탈출 조건 만족 이벤트가 발생하여, 탈출 조건 만족 이벤트를 구독한 탈출 판정 메소드가 결과 창을 출력한다.

### 5.2.2. 적 사망 처리

- 적 캐릭터 상태를 `enemyState=dead` (사망)으로 변경한다.
- 플레이어 추적 패턴을 해제하고 사망 애니메이션을 재생한다.
- 사망 상태로 변경된 시점에 전리품 스폰 처리를 실행한다.
    - 전리품 목록에 기재된 각 아이템을 적 처치 위치에 생성한다.
- 사망 애니메이션 재생 길이만큼 시간 경과 후 적 캐릭터 오브젝트를 비활성화하여 적 ObjectPool에 전달한다.

# 6. UI

## 6.0. 캔버스 설정

- 각 UI를 배치하는 Canvas Scaler는 이하와 같이 설정한다.
    - UI Scale Mode: Scale with Screen Size
    - Reference Resolution:  X 2400 × Y 1080
    (스마트폰 기기 표준에 가까우면서 가로 길이 값을 단순화하기 편한 20:9 비율)
    - Screen Match Mode: Extend
    - Reference Pixels per inch: 100

- 플레이어가 월드 신에 진입하여 게임 플레이 중에 출력하는 UI.

| No. | UI 요소 이름 | 동작 설명 |

# 7. 데이터 인덱스

## 7.1. MainPlayer

- 플레이어가 직접 조작하는 메인 캐릭터의 데이터.

| No. | Index | 자료형 | 데이터명 | 설명 |
| --- | --- | --- | --- | --- |
| 0 | TID | int | 고유 ID | 각 플레이어 캐릭터의 고유 ID |
| 1 | hpMax | int | 최대 체력 | 체력 능력치의 맵 입장 시 보유하는 최대 값.
  • 플레이어의 보유 체력이 0이 되면 맵 플레이가 중단(강제 각성)된다. |
| 2 | moveSpeed | float | 이동 속도 | 이동 명령을 입력받았을 때 정해진 시간 간격(기본 1초)마다 플레이어 캐릭터가 맵 위에서 이동하는 거리 값. |
| 3 | manaRegen | int | 초당 마나 회복 | 플레이어의 보유 마나가 1초마다 회복되는 수량 |
| 4 | roles | enum | 역할군 | 각 플레이어 캐릭터의 역할군 타입
  • breaker (브레이커): 침식체의 단단한 몽막을 빠르게 붕괴시키는 역할
  • striker (딜러): 몽막이 붕괴되어 드러난 심상핵을 직접 공격해 치명적인 타격을 주는 역할
  • supporter (서포터): 아군의 체력을 회복하거나 디버프를 해제하며 안전한 이동을 지원하는 역할
  • controller (컨트롤러): 침식체의 진형을 무너뜨리거나 행동 불능(스턴, 이속 감소)을 유도하는 역할
  • defender (디펜더): 방막을 형성하고 거점을 방어하며 아군의 안정성 회복 및 위기 생존을 지원하는 역할 |
| 5 | resonanceTag_role | enum | 역할 태그 | 심상 공명 시스템을 발동시킬 수 있는 역할 태그 타입 |
| 6 | resonanceTag_piece | enum | 기물 태그 | 심상 공명 시스템을 발동시킬 수 있는 기물 태그 타입 |
| 7 | resonanceTag_belong | enum | 소속 태그 | 심상 공명 시스템을 발동시킬 수 있는 소속 태그 타입 |

## 7.2. Enemy

- 적 캐릭터의 기본 데이터.

| No. | Index | 자료형 | 데이터명 | 설명 |
| --- | --- | --- | --- | --- |
| 0 | TID | int | 고유 ID | 각 메인 캐릭터의 고유 ID |
| 1 | hpMax | int | 최대 체력 | 체력 능력치의 맵 입장 시 보유하는 최대 값.
적의 보유 체력이 0이 되면 처치 처리가 발생한다. |
| 2 | def | int | 방어력 | 플레이어의 공격으로부터 잃는 체력 값을 정해진 수식에 따라 감소시켜주는 계수 값. |
| 3 | moveSpeed | float | 이동 속도 | 이동 명령을 입력받았을 때 정해진 시간 간격(기본 1초)마다 플레이어 캐릭터가 맵 위에서 이동하는 거리 값. |
| 4 | sightLength | float | 시야 거리 | 플레이어 찾기 루틴을 실행하는 영역의 거리(반지름) 값. |
| 5 | sightAngle | float | 시야 각도 | 플레이어 찾기 루틴을 실행하는 영역의 범위 각도 값.
  • 전방위에서 항상 감지하는 경우의 최대값은 360으로 입력 (원 deg 각도) |
| 6 | atk | int | 공격력 | 공격 루틴을 실행하여 플레이어 캐릭터의 보유 체력을 감소시키는 기본 값 |
| 7 | atkRate | float | 공격 간격 | 공격 루틴을 실행하는 시간 간격 값.
  • 공격 속도는 공격 간격의 역수 `inv(AtkRate)`로 계산한다. |
| 8 | atkRange | float | 공격 사거리 | 공격 루틴을 실행하는 영역의 거리(반지름) 값. |
| 9 | atkCooltime | float | 공격 쿨타임 | 공격 루틴이 시작된 시점부터 플레이어에게 피해 판정 이벤트를 입력하는 시점까지의 시간 간격 |
| 10 | dreamBarrier | int | 몽막 게이지 | 각 적 캐릭터가 보유한 몽막 게이지 최대 수치 |
| 11 | coreBonus | float | 심상핵 치명타 배율 | 적의 몽막 게이지가 0인 상태에서 플레이어의 공격으로 받는 추가 피해 |
| 12 | barrierRecoveryTime | float | 몽막 재생 시간 | 몽막 게이지가 0이 된 상태에서 몽막 게이지가 회복되기까지 걸리는 대기 시간 |
| 13 | rootGroup | int | 전리품 그룹 | 처치 시 전리품 생성 그룹 ID
  • [EnemyRoot] 테이블의 [RootGroup] 값에서 참조한다. |
| 14 | enemeModel | string | 적 모델링 | 적 캐릭터가 월드 신에 출력될 때 사용하는 프리팹 파일명 |
| 15 | baseAtkID | int | 일반 공격 ID | 적 캐릭터가 일반 공격으로 사용하는 스킬의 ID
  • [Skill] 테이블의 [TID] 값에서 참조한다. |
| 16 | skillID | list<int> | 스킬 ID | 적 캐릭터의 일반 공격 외 스킬 ID 리스트
  • [Skill] 테이블의 [TID] 값에서 참조한다. |

## 7.3. Skill

- 메인 캐릭터/지원 캐릭터 호출/적 캐릭터가 사용하는 스킬 데이터를 통합 관리하는 테이블.
    - effectType 및 effectValue는 2개 이상의 컬럼을 사용하여 1개 스킬에 여러 가지 효과를 부여한다.

| No. | Index | 자료형 | 데이터명 | 설명 |
| --- | --- | --- | --- | --- |
| 0 | TID | int | 고유 ID | 각 스킬의 고유 ID |
| 1 | skillUser | enum | 스킬 사용자 | 각 스킬의 사용자 종류 타입
  • main: 플레이어 메인 캐릭터
  • strike: 플레이어 지원 캐릭터
  • enemy: 적 캐릭터 |
| 2 | skillType | enum | 스킬 종류 | 각 스킬의 스킬 종류 타입
  • base: 기본 공격
  • passive: 패시브 스킬
  • manual: 수동 액티브 스킬
  • auto: 자동 액티브 스킬 |
| 3 | skillName | string | 스킬 이름 | 각 스킬의 이름 스트링 |
| 4 | skillIcon | string | 스킬 아이콘 | 각 스킬의 아이콘 파일명 |
| 5 | skillCooltime | float | 스킬 쿨타임 | 각 스킬 사용 후 다음 사용까지 재사용 대기시간 |
| 6 | effectType | enum | 스킬 효과 종류 | 각 스킬의 효과 종류 타입
  • damage: 보유 체력 피해
  • healHP: 보유 체력 회복
  • leap: 도약,돌진 |
| 7 | effectValue | string | 스킬 효과 값 | 각 스킬의 효과 종류별 적용 값
  • `effectType=damage`: 공격력 배율(float 변환)
  • `effectType=heal`: 보유 체력 회복량(int 변환)
  • `effectType=leap`: 돌진 거리(float 변환) |

- 적 캐릭터 처치 시 생성되는 전리품 중 회수품에 해당되는 아이템의 데이터.

| No. | Index | 자료형 | 데이터명 | 설명 |
| --- | --- | --- | --- | --- |
| 0 | TID | int | 고유 ID | 회수품 아이템의 고유 ID |
| 1 | itemName | string | 아이템 이름 | 각 회수품 아이템의 이름 스트링 |
| 2 | itemIcon | string | 아이템 아이콘 | 각 회수품 아이템의 UI 출력 아이콘 파일명 |
| 3 | itemDesc | string | 아이템 설명 | 각 회수품 아이템의 상세 설명 스트링 |
| 4 | itemCategory | enum | 아이템 종류 | 각 회수품 아이템의 종류 타입

  • armor: 방어구
  • consume: 소비 기물 |

  • `category=idle` (기타 아이템): 0으로 입력한다.
  • `category=weapon` (무장): [Weapon] 테이블의 [TID] 값에서 참조한다.
  • `category=armor` (방어구): [Armor] 테이블의 [TID] 값에서 참조한다.
  • `category=consume` (소비 기물): [Consume] 테이블의 [TID] 값에서 참조한다. |
| 6 | size | int[2] | 슬롯 크기 | 각 회수품 아이템을 인벤토리에 보관 시 차지하는 슬롯 크기
  • {가로, 세로} 순서로 입력
  • 각성 보존 슬롯 보관 시 차지하는 슬롯 개수는 `가로 × 세로` 로 계산한다. |
