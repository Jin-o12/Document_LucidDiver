## 0. 프로토타입 목표

이하의 플레이 최소 루프가 구현 가능한지 검증한다.

- 플레이어가 2.5D 맵 위에서 움직인다.
- 기본 공격이 나간다.
- 적 1종이 플레이어를 따라온다.
- 아이템 1종을 주울 수 있다.
- 탈출 지점에 닿으면 결과창이 뜬다.

## 1. 플레이어 캐릭터 (Player Character)

### 1.1 오브젝트 구성
플레이어 캐릭터 오브젝트는 이하의 하위 오브젝트로 구성된다.

- **위치 기준점 (Transform)**: 플레이어의 위치 및 방향(Y 위, Z 정면) 정보를 계산하는 기준점.
- **몸통 (Collider)**: 플레이어↔적, 플레이어↔장애물 간의 접촉 상호작용 발동 여부를 판정하는 기준 영역.
  - 장애물 판정을 위해 Trigger 설정을 비활성화한다.
- **상호작용 구역 (Collider)**: 플레이어↔상호작용 가능 오브젝트(무장, 회수품) 간의 상호작용 UI 출력 여부를 판정하는 기준 영역.
  - Trigger 설정을 활성화하여 사용한다.
- **손 (Transform)**: 무장 아이템 장착 시 무장 오브젝트가 배치되는 위치 기준점.
  - **공격 방향 (Vector3)**: 위치 기준점을 시작점, 손을 끝점으로 하는 방향 벡터.
    - 높이(Unity 좌표계의 Y축) 값을 0으로 초기화한 후 길이 Normalize를 적용하여 사용한다.

### 1.2 플레이어 상태
플레이어 캐릭터는 bool 또는 enum으로 표기되는 상태 값을 갖는다.

- **재장전 중 (`isReloading` : bool)**: 플레이어가 탄창을 전부 사용하여 재장전 상태인지 확인.
- **생존 상태 (`livingState` : enum)**: 플레이어의 메인 캐릭터 상태를 확인.
  - `0`: idle `기본`
  - `1`: escape `탈출 중`
  - `2`: gameover `강제각성`
  - 탈출 조건 우선 순위는 enum 오름차순으로 판정한다(탈출 중 → 강제각성 순서로 처리).

### 1.3 플레이어 조작

#### 1.3.1 이동 (Movement)
- 화면 왼쪽의 가상 조이스틱을 드래그하여 이동 명령을 입력한다.
  - PC 환경(Unity Editor 및 에뮬레이터)에서는 Input System을 적용하여 WASD 및 화살표(`↑←↓→`) 조작을 사용해 이동할 수 있게 한다.
- 이동 명령 값은 이동 가상 조이스틱의 드래그 시작 지점과 드래그 유지 지점 사이의 방향 벡터(Vector2, 길이 Normalize 적용)을 전달한다.
- 입력된 이동 명령 값에 플레이어의 이동 속도 및 이동 명령 입력 시간(`Time.deltaTime`)을 곱한 값만큼 플레이어 캐릭터의 실제 이동을 처리한다.

#### 1.3.2 기본 공격 (Basic Attack)
- 화면 오른쪽의 가상 조이스틱을 드래그하여 기본 공격 명령을 입력한다.
  - 무장 아이템을 획득하지 않은 상태에서는 기본 공격 명령이 입력되지 않는다.
  - `재장전 중 == true` 상태인 동안에는 기본 공격 명령이 입력되지 않는다.
- 기본 공격 명령 값은 기본 공격 가상 조이스틱의 드래그 시작 지점과 드래그 유지 지점 사이의 방향 벡터(Vector2, 길이 Normalize 적용)을 전달한다.
  - PC 환경에서는 UI Raycast Target이 적용되지 않는 영역을 마우스 왼쪽 버튼 클릭으로 홀드하여 플레이어 위치 기준점의 UI Canvas 상 위치와 클릭 지점 사이의 방향 벡터를 기본 공격 명령 값으로 입력할 수 있도록 한다.
- 입력된 기본 공격 명령 벡터를 높이 값이 0인 Vector3으로 환산한 후, 플레이어 캐릭터의 공격 방향 벡터를 기본 공격 명령 벡터 방향으로 회전시킨다.
- 플레이어 캐릭터 오브젝트의 위치 기준점에서 공격 방향 벡터 방향으로 무장의 발사 가능 사거리 길이만큼 공격 Ray를 출력한다.
- 공격 Ray가 적의 피격 범위 Collider에 접촉하면 무장 아이템 오브젝트의 총구 위치에서 총알 발사 루틴을 실행한다.
- 총알 발사 루틴 1회 실행 후 발사 간격 시간 동안 총알 발사 루틴을 실행하지 않고 대기한다.

#### 1.3.3 오브젝트 상호작용
- 플레이어 캐릭터의 상호작용 구역 Collider에 상호작용 가능한 오브젝트의 Collider가 접촉하면 상호작용 버튼이 오브젝트의 정해진 위치에 출력된다.
- 출력된 상호작용 버튼을 터치(PC 환경에서는 마우스 클릭)하여 상호작용을 실행한다.

## 2. 맵 설정 (Map & Camera)

본 게임의 맵은 2.5D 시점으로 작업한다.
- 3D 신에서 X, Z의 2개 좌표만을 사용하여 캐릭터(플레이어, 적)이 이동하는 레벨을 구성한다.

*※ 카메라 시점 벤치마킹 예시 레퍼런스: VOID DIVER*

### 2.1 카메라 설정
- 메인 카메라에 `Cinemachine`을 적용하여, 플레이어 메인 캐릭터 Transform을 **Follow** 및 **LookAt** 대상으로 지정한다.
  - 카메라가 플레이어의 위치 이동을 추적하여 동일한 속도로 이동한다.

### 2.2 시작 지점
- 각 맵마다 정해진 위치에 시작 지점을 설정한다.
  - 시작 지점에는 플레이어 캐릭터 오브젝트를 출력하는 출력 기준점 Transform 오브젝트가 배치된다.
- 플레이어가 맵에 진입하면 플레이어 캐릭터 오브젝트가 시작 지점에 출력된다.
  - 시작 지점의 출력 기준점 위치와 플레이어 캐릭터의 위치 기준점이 일치하도록 출력한다.

### 2.3 탈출 지점
- 각 맵에 정해진 개수의 탈출 지점을 설정한다.
  - 탈출 지점에는 플레이어 캐릭터 오브젝트의 탈출 여부를 감지하는 탈출 구역 (Collider) 오브젝트가 배치된다.
    - 탈출 구역 Collider의 Trigger 설정을 활성화한다.
  - 탈출 지점에 데이터 테이블과 연동되는 스크립트를 통해 탈출 가능 조건 및 탈출 가능한 플레이어 수를 설정한다.
- 플레이어의 몸통 Collider와 탈출 지점의 탈출 구역 Collider가 접촉하면 `OnTriggerEnter` 이벤트를 통해 플레이어 탈출 판정을 실행한다.

### 2.4 탈출 판정
- 맵 플레이 중 이하의 어느 조건 중 1개 이상이 만족되면 탈출 판정을 실행한다.
  - 플레이어 캐릭터의 보유 체력이 0이 됨 → `강제 각성` 상태가 된다.
  - 플레이어가 탈출 지점에 도착 → `탈출 중` 상태가 된다.
- 탈출 판정 조건은 각 플레이어마다 개별로 계산한다.
- 탈출 판정 실행 시 탈출 조건이 만족된 시점의 보유 체력, 처치한 적 수, 플레이 시간, 인벤토리 리스트를 저장하여 결과 창에 출력한다.

## 3. 아이템 시스템 (Items)

### 3.1 아이템 생성
- 월드 신이 생성되는 시점에 월드 신마다 지정된 위치에 아이템이 생성된다.

### 3.2 무장 (Weapon)
플레이어가 장착하여 적을 공격하는 데에 사용하는 아이템.

#### 3.2.1 오브젝트 구성
무장 아이템 오브젝트는 이하의 하위 오브젝트로 구성된다.

- **손잡이 (Transform)**: 플레이어 캐릭터가 무장 장착 시 플레이어 캐릭터의 손 오브젝트와 무장 아이템 오브젝트의 위치 및 방향을 일치시키는 기준점.
- **총구 (Transform)**: 총알 발사 루틴이 출력되는 위치 및 방향 기준점.

총알 아이템 오브젝트는 이하의 하위 오브젝트로 구성된다.

- **탄두 (Collider)**: 총알 오브젝트와 적 캐릭터의 충돌 판정을 감지하는 영역.
  - Trigger 설정은 비활성화하고 초기 개발을 진행한다. (MVP 개발 중 관통 시스템 적용)

#### 3.2.2 무장 획득
- 월드 맵에 생성된 무장 아이템은 플레이어가 오브젝트 상호작용을 통해 획득한다.
- 무장 아이템 획득 시 이하의 순서에 따라 획득 여부를 결정한다.
  - 플레이어가 장착하고 있는 무장이 없는 경우(무장 슬롯 `null`): 상호작용한 무장 아이템을 빈 무장 슬롯에 장착한다.
    - 보유 총알 수는 무장 아이템을 획득한 시점에 총 총알 수와 같은 값이 된 후, 현재 탄창 수만큼 감소한다.
  - 장착하고 있는 무장이 있는 경우(무장 슬롯 `a`)
    - 상호작용한 무장이 장착하고 있는 무장과 같으면(획득 무장 `a`): 무장 슬롯에 장착하고 있는 무장의 보유 총알 수를 그 무장 아이템의 총 총알 수와 같은 값으로 갱신한다. (현재 탄창 수만큼 감소하는 처리는 실행하지 않음)
    - 상호작용한 무장이 장착하고 있는 무장과 다르면(획득 무장 `b`): 무장 슬롯에 장착하고 있는 무장을 `b`로 교체하여 새로 장착한다.
      - 기존에 장착하고 있던 무장 `a`는 제거된다.
      - 새로 장착한 무장 `b`의 보유 총알 수는 무장 아이템을 획득한 시점에 무장 `b`의 총알 수와 같은 값이 된 후, 현재 탄창 수만큼 감소한다.

#### 3.2.3 총알 발사
- 총알 발사 루틴 실행 시 총알 오브젝트가 총구 위치에서 공격 방향 벡터 방향으로 정렬되어 출력된다.
  - 총알 Object Pool에서 무장 오브젝트로 추출 후 활성화하여 `OnEnable` 메소드부터 실행한다.
- 출력된 총알 오브젝트는 비행 시간만큼 유지되며, 비행 속도 값에 따라 공격 방향 벡터 방향으로 이동한다.
- 총알 오브젝트는 비행 시간이 경과하거나 적, 장애물 Collider에 접촉하면 비활성화되어 총알 Object Pool로 되돌아간다.
  - 비행 시간 경과 시 별도의 처리 없음.
  - 총알 탄두 Collider가 적 몸통 Collider에 접촉 시 적 명중 판정 처리를 실행.
  - 총알 탄두 Collider가 장애물 Collider에 접촉 시 장애물 명중 판정 처리를 실행.

#### 3.2.4 재장전
- 총알 발사는 정해진 탄창 용량 수치만큼만 연속으로 실행할 수 있다.
  - 현재 탄창(int)과 보유 총알 수(int), 재장전 중(bool)을 내부 변수로 지정한다.
  - 총알 발사 루틴을 실행할 때마다 그 횟수만큼 현재 탄창 값이 감소한다.
- 현재 탄창이 0이 된 시점에 이하의 순서에 따라 재장전 루틴을 실행한다.
  1. `재장전 중 = true` 상태가 된다.
  2. 재장전 시간만큼 대기한다.
  3. 보유 총알 수를 최대 탄창만큼 감소시키고, 실제로 감소시킨 값(A)을 내부 변수에 저장한다.
    - 보유 총알 수가 최대 탄창 미만이었을 경우에 대비하기 위해 별도의 변수를 사용함.
    - 보유 총알 수가 0 이하인 경우에는 재장전 루틴이 더 이상 진행되지 않고 강제 종료된다.
  4. 현재 탄창 값을 A로 설정한다.
  5. `재장전 중 = false` 상태가 된다.

### 3.3 회수품 (Loot)
- 회수품은 월드 내에 생성되는 아이템 중 장비 이외의 아이템을 가리킨다.
- 회수품 아이템과 플레이어가 상호작용하면 해당 플레이어의 인벤토리에 상호작용한 회수품을 등록한다.
  - 인벤토리 등록 처리 후 월드 맵의 회수품 오브젝트를 비활성화하여 아이템 Object Pool로 되돌린다.

## 4. 적 (Enemy)

### 4.1 오브젝트 구성
적 캐릭터 오브젝트는 이하의 하위 오브젝트로 구성된다.

- **위치 기준점 (Transform)**: 적의 위치 및 방향(Y 위, Z 정면) 정보를 계산하는 기준점.
- **몸통 (Collider)**: 적과 다른 구성 요소 사이의 접촉 상호작용 발동 여부를 판정하는 기준 영역.
  - 장애물 판정을 위해 Trigger 설정을 비활성화한다.

### 4.2 플레이어 추적
적 캐릭터 오브젝트에 플레이어 추적 패턴을 설정한다.

- 플레이어 추적 패턴은 시야 거리만큼의 반지름과 시야 각도만큼의 사이각을 가지는 부채꼴 영역(이하 `추적 영역`) 내에서 작동한다.
  - 추적 영역은 위치 기준점 정면 방향에서 좌우(각각 -, + 방향)로 시야 각도 값의 절반만큼 벌린 영역이 된다.
- 적 캐릭터의 추적 영역 내에 플레이어가 진입하면 적의 추적 상태가 `추적 중`으로 변경된다.
  - 플레이어 진입은 플레이어의 위치 기준점이 추적 영역 내에 위치하는지 여부로 판정한다.
  - `강제 각성` 상태나 `탈출 중` 상태인 플레이어는 진입 판정을 실행하지 않는다.
- `추적 중` 상태인 적 캐릭터는 추적 영역 내의 플레이어 중 거리가 가장 가까운 캐릭터를 향해 이동한다.
  - 적의 위치 기준점 정면 방향을 플레이어 위치 방향으로 Rotate시킨다. (보간 적용)
- 추적 영역 내에 적 캐릭터와의 거리 값이 공격 사거리 이하인 플레이어 캐릭터가 있으면 적의 추적 상태가 `공격 개시`로 변경된다.
  - `공격 개시` 상태인 적은 위치 기준점 사이의 거리가 가장 가까운 플레이어 1명에게 공격 루틴을 실행한다.
    - 공격 애니메이션을 재생하고, 피해 판정 이벤트가 입력된 시점에 공격 대상 플레이어에게 체력 감소 처리를 발생시킨다.
  - 공격 루틴 실행 후 공격 간격 시간 동안 대기한다.
    - 대기 중 추적 영역 내의 플레이어 캐릭터를 검색하여 추적 상태를 갱신한다.

적 캐릭터의 추적 패턴은 이하의 상태에 따라 제어한다.

- **생존 상태 (`isAlive` : bool)**: 현재 생존 중인지 확인.
- **추적 상태 (`chaseState` : enum)**: 플레이어 추적 패턴의 작동 상태를 확인.
  - `idle` `기본`
  - `chase` `추적 중`
  - `attack` `공격 개시`
  - 추적 상태 우선 순위는 enum 내림차순으로 판정한다(공격 개시 → 추적 중 → 기본 상태 순서로 처리).

## 5. 전투 (Combat)

### 5.1 피해 계산
플레이어 및 적 캐릭터는 보유 체력(int)을 내부 변수로 가진다.

- 보유 체력은 각 캐릭터가 월드에 등장한 시점에 최대 체력 값만큼의 초기값을 부여받는다.
- 플레이어가 발사한 총알이 적에게 명중 판정 발생 시 적의 보유 체력이 감소한다.
- 적이 공격 루틴을 실행하여 플레이어의 보유 체력을 감소시킨다.
- 공격받은 대상의 보유 체력이 감소하는 수량은 이하의 수식을 사용하여 계산한다.

$$체력감소_B = 공격력_A \div (방어도\ 상수 + 방어력_B)$$

- $A$: 공격한 대상, $B$: 공격받은 대상.
- 기본 공격은 [공격 배율]을 100%로 계산한다.
- 방어도 상수 값은 밸런스 기획 단계에서 결정한다.

### 5.2 체력 소진 시 처리
공격받은 대상의 보유 체력이 0 이하가 되면 대상의 종류에 따라 사망 처리를 실행한다.

#### 5.2.1 플레이어 - 강제 각성
- 플레이어의 `livingState` 상태를 `gameover` `강제 각성`으로 변경한다.
- 탈출 조건 만족 이벤트가 발생하여, 탈출 조건 만족 이벤트를 구독한 탈출 판정 메소드가 결과 창을 출력한다.

#### 5.2.2 적 - 전리품 생성
- 적의 `isAlive` 상태를 `false` `처치됨`으로 변경한다.
- 처치 이벤트가 발생한다.
  - 전리품 생성 메소드를 처리한다.
  - 적 오브젝트 비활성화 및 적 Object Pool 전달을 처리한다.
