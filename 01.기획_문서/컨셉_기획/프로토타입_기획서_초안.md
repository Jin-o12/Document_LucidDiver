# 🎮 《침몽도시: 루시드 다이버》 프로토타입 기획서 초안

## 0. 기획 의도 (Design Intent)
- **목적**: 《침몽도시: 루시드 다이버》의 1차 프로토타입 구현 범위를 정의하고 핵심 플레이 루프를 단순하고 명확하게 구축하여 조작성 및 기본 전투 안정성을 검증합니다.
- **핵심 가치**: 전체 시스템(멀티플레이, 영구 보존 슬롯, 채널링 잠금 해제, 소음 어그로 등)을 구현하기 전, 플레이어가 2.5D 전장에 진입하여 기본 조작을 수행하고, 적을 조우 및 처치한 후, 필드의 아이템을 회수하여 탈출하는 **최소 플레이 루프 (Minimum Play Loop)**의 재미와 물리적 구동 안정성을 최우선 확보합니다.

---

## 1. 개요 (Overview)
- **기능명**: 루시드 다이버 1차 프로토타입 기획 (최소 플레이 루프 검증)
- **담당자**: 기획팀 / Antigravity
- **우선순위**: 상 (High)
- **현재 상태**: 초안 (Draft)
- **핵심 목표**: 진입 ➔ 이동 ➔ 전투 ➔ 아이템 회수(즉시 획득) ➔ 탈출(즉시 탈출) ➔ 결과 확인의 6단계 루프 검증.

---

## 2. 상세 로직 및 프로세스 (Core Logic)

### 2.1 핵심 플레이 루프
- **트리거**: 플레이어가 로비/결과 화면에서 [시작]을 선택해 테스트 맵에 진입할 때 실행됩니다.
- **프로세스 시퀀스**:

```mermaid
flowchart TD
    Start([1. 테스트 맵 진입]) --> Move[2. 2.5D 캐릭터 이동 조작]
    Move --> Detect[3. 적 조우 및 추적 발생]
    Detect --> Attack[4. 희미한 잔상 에너지 권총 사격]
    Attack --> HitCheck{기본 공격 적중?}
    HitCheck -->|Yes| Hit[5. 적 피격 반응 및 체력 감소]
    HitCheck -->|No| Move
    Hit --> Loot[6. 테스트 심상기물 회수<br>즉시 획득]
    Loot --> Escape[7. 탈출 전화부스 이동 및 접촉]
    Escape --> Result([8. 결과창 출력 및 보상 정산 확인])
```

### 2.2 몬스터 AI 상태 처리
- **트리거**: 몬스터가 활성화된 상태에서 상시 실행되는 상태 머신(FSM)입니다.
- **AI 시퀀스**:

```mermaid
flowchart TD
    Idle[순찰 상태: 노드 2개 왕복] -->|플레이어 감지 반경 진입| Detect[플레이어 추적]
    Detect -->|공격 범위 도달| Attack[근접 돌진 공격 실행]
    Attack -->|피격 감지| Hit[피격 경직 / 체력 차감]
    Hit -->|체력 > 0| Detect
    Hit -->|체력 <= 0| Dead[비활성화 / 사망 및 기물 드롭]
```

### 2.3 주요 세부 로직
1. **플레이어 이동 및 공격**: 가상 조이스틱 또는 키보드 방향키를 통해 2.5D 평면 상에서 움직이며, 사격 버튼 입력 시 지정한 방향으로 기본 무장인 **'희미한 잔상 (REM Tracker)'**의 투사체 탄환을 발사합니다.
2. **아이템 회수**: 
   - 필드 내 배치되거나 몬스터 처치 시 드롭되는 테스트용 심상기물 오브젝트에 플레이어가 접촉(또는 F키 입력)하면 즉시 `acquiredItems` 데이터에 기록되고 해당 오브젝트가 비활성화됩니다.
   - *※ 기획서상의 채널링(1.5초) 및 소음 핑 등의 어뷰징 차단 기믹은 프로토타입 구현 범위에서 제외하고 즉시 획득으로 처리합니다.*
3. **세션 탈출**: 활성화된 공중전화부스(`Exit_Point`) 영역에 캐릭터의 Collider가 접촉하면 실시간 전투 세션이 즉시 성공 종료되고 결과 UI창이 활성화됩니다.
   - *※ 3초 탈출 채널링(피격 캔슬) 기믹은 프로토타입 구현 범위에서 제외하고 즉시 탈출로 처리합니다.*

## 3. 데이터 명세 (Data Specification)

### 3.1 세션 상태 제어 데이터
| No. | 필드명 | 타입 | 설명 | 기본값 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | `playerHp` | `int` | 플레이어 캐릭터의 현재 체력 수치 | 100 |
| 2 | `monsterHp` | `int` | 테스트 몬스터의 현재 체력 수치 | 50 |
| 3 | `acquiredItems` | `list<string>` | 현재 세션에서 회수 완료된 아이템 리스트 | `[]` |
| 4 | `isEscaped` | `bool` | 탈출 성공 여부를 판정하는 상태 값 | `false` |

### 3.2 프로토타입 등장 요소 목록
| 구분 | 명칭 | 변수명/ID | 역할 및 설명 |
| :--- | :--- | :--- | :--- |
| **플레이어** | 루시드 다이버 테스트 캐릭터 | `Player_Test` | 2.5D 이동, 기본 무장 '희미한 잔상' 사격, 아이템 회수(즉시), 탈출부스 접촉 탈출 |
| **몬스터** | 테스트 침식체 (순찰 잔몽체) | `Monster_Test` | 순찰 노드를 돌다 플레이어를 감지 및 추적하며 물리적 대치 제공 |
| **아이템** | 테스트 심상기물 | `Item_Test` | 필드 스폰되거나 몬스터 사망 시 드롭되며, 접촉 즉시 획득되어 결과에 반영되는 기물 |
| **탈출지점** | 탈출 지점 오브젝트 | `Exit_Point` | 세션을 즉시 종료시키고 결과창 UI를 호출하는 전화부스 (접촉 시 즉시 탈출) |

## 4. UI/UX 흐름

### 4.1 프로토타입 HUD
- **체력 UI**: 화면 좌측 상단에 `Player_Test`의 `playerHp`를 바(Bar) 형태로 표시합니다.
- **아이템 UI**: 화면 우측 상단에 현재까지 `acquiredItems`에 담긴 개수를 표시합니다. (예: `획득한 기물: 0개`)
- **조작 UI**: 모바일 환경에 대응하기 위해 가상 조이스틱과 공격/상호작용 가상 버튼을 화면 하단 양측에 배치합니다.
- *※ 채널링 진행도를 표시하는 게이지 바(채널링 UI)는 프로토타입 구현 범위에서 제외됩니다.*

### 4.2 결과 화면 (Result UI)
- 세션 종료(`isEscaped = true`) 또는 플레이어 체력이 0이 되어 세션이 종료될 시 팝업 형태로 화면 중앙에 활성화됩니다.
- **결과 메시지**:
  - 탈출 성공 시: "탈출 성공!" 문구 및 회수된 아이템 목록(`acquiredItems`)을 출력합니다.
  - 탈출 실패(체력 0) 시: "탈출 실패 (강제 각성)" 문구 및 일반 획득 아이템이 소실되었음을 보여줍니다.
- **동작 제어**:
  - `다시 시작(Retry)` 버튼: 현재 씬을 재로드하여 프로토타입 플레이 루프를 처음부터 다시 진행합니다.
  - `종료(Exit)` 버튼: 테스트 씬을 종료하고 로비 또는 프로그램 종료를 수행합니다.

## 6. 주의 사항 및 제약 (Constraints)
- **복잡한 인터랙션 제외**: 프로토타입 범위 내에서는 1.5초 기물 해제 채널링, 3초 탈출 채널링, 소음 핑, 제한시간(초조함/심층 디버프) 시스템, 각성 보존 슬롯 및 오퍼레이터 시스템 등 복잡한 시스템 기믹은 모두 배제하고 즉시 획득 및 즉시 탈출로 단순화합니다.
- **멀티플레이 제외**: 프로토타입 범위 내에서는 1인 로컬 플레이만 지원합니다. 2인 협동 등 멀티플레이 동기화는 최종 MVP에서 구현하고 테스트합니다.
- **임시 리소스 사용**: 그래픽 및 사운드 에셋은 개발 검증용 임시(Placeholder) 리소스를 우선 사용합니다.
